What is git?

It is a system that monitors the changes you make on a document (software project, office document, etc.) step by step, saves it whenever you want, and allows you to store and manage it on a computer on the internet or on a local device (respository / repo / repository).

You may have also heard the term Source Code Management System instead of Version Control System. Both mean the same thing.

What Does GIT Give Us?

  • You can store your files and version control information in multiple places (distributed). Thus, you can access your files independently of the device.
  • With the SnapShot (snapshot) feature, you can record the current state of the project or files at any time by "commit". This way, if you encounter a bug in the future, you can revert to any version at any time.
  • After SnapShot is taken, you can see the modified files.
  • It allows teams to work together on the same project. In this way, you can facilitate project development processes with the community.
  • You can specify files that you do not want to be versioned in the project. (like node_modules, .mp4, .log, .env)

Some important terms

Some terms that we need to know;

  • untracked: It refers to newly created files that are not yet tracked by GIT.
  • unstaged: It refers to files that have been updated but not prepared for committing.
  • staged: Indicates files that are ready to be committed.
  • deleted: It refers to files that have been deleted from the project but not removed via GIT.

Basic commands

It is used to create an empty git repository in the directory of a project that is not yet under version control.

$ git init

GIT has many configurations and settings, two of which are user.name and user.email. We use the following commands to configure these settings.

$ git config --global user.name "Name Surname" $ git config --global user.email "test@email.com"

It is used to send newly added or modified files to the staged environment.

$ git add

It is a command that allows to untrack a file added to the Staged environment, that is, to make it untracked.

$ git rm --cached

It gives information about the current status of the project being worked on. Information such as changes made, added and deleted files are listed.

$ git status

Commit is sending the files taken to the staged environment to the Local Repository. A best practice is to include a message describing the changes made during each save. In addition, each commit has a unique ID. In this way, you can revert to an old commit and you will not have any loss.

$ git commit -m "first commit message"

It allows us to view the commit history in the project. All commits are listed with their id, author, date and message.

$ git log

It is used to add, delete or list a new branch on local or remote repository.

$ git branch

It is used when we want to switch between branches or commits.

$ git checkout

It is used when we want to merge the changes in another branch with the current branch.

$ git merge

It enables a copy of the files in an existing Remote Repository to be created on your computer.

$ git clone

It sends the commits we receive in our project to the remote repository.

$ git push origin master

It shows the differences between the files after the changes made on the repository.

$ git diff HEAD

What is a gitignore file?

The .gitignore file is a plain text file created in the root directory of our project. As the name suggests, it says ignore me. More precisely, if you have files that you want to ignore, that you do not want to be tracked in your local workspace, that are not needed for your teammates, or if the size of these files is so large that you do not need to put them in your repo, it says, "Use me."

So what are these types of files?:

  • Dependencies downloaded from package manager,
  • your image and video files (file sizes may be too large)
  • IDE plugins (eg .vscode)
  • Your files (configurations for your database) that should only be in your own workspace and not visible to others
  • Files containing API keys, credentials, or sensitive information (.env)
  • Temporary files in your working directory
  • Log files
  • Useless system files (for example the .DS_Store file of the MacOS operating system)
  • Files created like dist

Or you can have any file.
